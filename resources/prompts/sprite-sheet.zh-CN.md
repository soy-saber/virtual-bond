# Virtual Bond 序列帧皮肤生成 Prompt 模板

模板版本：`1`

用途：生成符合 Virtual Bond 皮肤规范的规则网格 Sprite Sheet。使用前请替换所有 `{{变量}}`。不同图片模型对精确尺寸、透明背景和严格网格的遵循程度不同，生成结果必须经过切帧与布局校验后才能作为正式皮肤资源。

## 变量

| 变量             | 含义                       | 示例                               |
| ---------------- | -------------------------- | ---------------------------------- |
| `{{角色描述}}`   | 外貌、服装、配色和辨识特征 | 银灰短发、紫色风衣、金色发夹       |
| `{{参考图说明}}` | 角色参考图的使用要求       | 严格保持参考图中的脸型、发型和服装 |
| `{{动作名称}}`   | 清单中的动作键名           | `walk`                             |
| `{{动作描述}}`   | 本次动画的完整动作         | 放松、缓慢地向前行走               |
| `{{画面风格}}`   | 像素、手绘、卡通等视觉风格 | 精细二次元游戏立绘风格             |
| `{{帧数}}`       | 有效动画帧总数             | `8`                                |
| `{{列数}}`       | 网格列数                   | `4`                                |
| `{{行数}}`       | 网格行数                   | `2`                                |
| `{{单帧宽度}}`   | 每个单元格宽度             | `256`                              |
| `{{单帧高度}}`   | 每个单元格高度             | `256`                              |
| `{{总宽度}}`     | 完整 Sheet 宽度            | `1024`                             |
| `{{总高度}}`     | 完整 Sheet 高度            | `512`                              |
| `{{锚点X}}`      | 单帧内脚底锚点横坐标       | `128`                              |
| `{{锚点Y}}`      | 单帧内脚底锚点纵坐标       | `240`                              |
| `{{朝向}}`       | 角色面向                   | 右侧                               |
| `{{循环要求}}`   | 循环或一次性动作要求       | 首尾自然衔接的无缝循环             |
| `{{起始姿势}}`   | 第一帧姿势                 | 双脚自然站立，重心居中             |
| `{{结束姿势}}`   | 最后一帧姿势               | 回到接近第一帧的站立姿势           |
| `{{逐帧变化}}`   | 按播放顺序描述每一帧       | 见下方示例                         |

## 正向 Prompt

```text
请生成一张可直接用于桌面宠物动画的专业 Sprite Sheet 序列帧贴图。

【角色】
{{角色描述}}

【参考图要求】
{{参考图说明}}

【动作】
动作名称：{{动作名称}}
动作内容：{{动作描述}}
角色朝向：{{朝向}}
循环方式：{{循环要求}}

【画面风格】
{{画面风格}}

【序列帧布局】
- 必须生成且只生成 {{帧数}} 个有效动画帧。
- 使用严格的 {{列数}} 列 × {{行数}} 行规则网格排列。
- 每个单元格必须是 {{单帧宽度}} × {{单帧高度}} 像素。
- 完整图片必须是 {{总宽度}} × {{总高度}} 像素。
- 播放顺序为从左到右，再从上到下。
- 所有单元格尺寸完全一致，不得合并、重叠或错位。
- 如果网格单元格数量多于有效帧数，多余单元格必须保持完全透明。
- 单元格之间不需要边框、编号、文字、分隔线或额外间距。

【角色一致性】
- 每一帧必须保持完全相同的角色身份、脸型、发型、服装、配色和身体比例。
- 每一帧必须使用相同的镜头角度、透视、角色大小和绘制精度。
- 不得在不同帧中改变服装结构、装饰数量、发型长度或身体比例。
- 每帧只允许出现同一个角色，不得出现分身、残影或额外角色。
- 角色必须完整位于单元格内，头发、手、脚、衣摆和道具不得被裁切。

【位置与锚点】
- 每一帧的脚底基准点必须固定在单元格坐标（{{锚点X}}, {{锚点Y}}）。
- 动作过程中角色可以改变姿势，但整体位置、身体比例和视觉重心不能无故漂移。
- 除动作明确要求离地外，脚底必须稳定接触同一条地面基准线。
- 所有帧必须保留一致的角色外边距。

【背景与输出】
- 背景必须完全透明，保留清晰、干净的透明通道。
- 不要绘制场景、地面、平台、光环、投影、边框、文字、水印或界面元素。
- 不要添加延伸到相邻单元格的光效、阴影或粒子。
- 边缘应清晰，适合切分成独立动画帧。

【动作过程】
第一帧：{{起始姿势}}
最后一帧：{{结束姿势}}

逐帧变化：
{{逐帧变化}}

请只输出最终 Sprite Sheet 图片，不要输出说明文字、示意图、动作名称或网格标记。
```

## 负面约束

如果图片模型支持单独的 Negative Prompt，使用以下内容；否则追加到正向 Prompt 末尾：

```text
不要出现：角色身份变化、脸型变化、发型变化、服装变化、颜色漂移、身体比例变化、不同镜头角度、不同角色尺寸、脚底锚点漂移、角色位置漂移、身体被裁切、头发被裁切、手脚缺失、肢体数量错误、多余角色、分身、残影、重复帧、缺失帧、错误动作顺序、帧之间融合、单元格重叠、网格错位、边框、编号、文字、标题、水印、分隔线、背景场景、地面、平台、不透明背景、跨单元格阴影、跨单元格光效、模糊边缘、压缩噪点。
```

## 逐帧变化示例：八帧行走循环

```text
第 1 帧：双脚自然站立，身体处于中间高度，准备迈步。
第 2 帧：左脚抬起并略微向前，右脚支撑，身体轻微下降。
第 3 帧：左脚向前伸出接近落地，双臂自然反向摆动。
第 4 帧：重心移动到左脚，身体处于最高点，右脚准备离地。
第 5 帧：双脚经过交叉位置，身体回到中间高度。
第 6 帧：右脚抬起并略微向前，左脚支撑，身体轻微下降。
第 7 帧：右脚向前伸出接近落地，双臂自然反向摆动。
第 8 帧：重心回正并接近第 1 帧姿势，确保循环播放时自然衔接。
```

## 生成后必须校验

1. 图片实际尺寸等于 `{{总宽度}} × {{总高度}}`。
2. 网格容量不小于 `{{帧数}}`。
3. 每个单元格能按固定尺寸无损切分。
4. 有效帧数量正确，多余单元格透明。
5. 每帧角色脚底锚点一致。
6. 角色没有跨越单元格边界。
7. 背景透明且没有跨帧阴影、光效或残留像素。
8. 按从左到右、从上到下播放时动作顺序正确。
9. 循环动作的首尾姿势能够自然衔接。
10. 将最终布局参数写入皮肤的 `manifest.json`。

## 实践建议

- 优先使用 `4×2`、`4×3` 等接近方形的网格，通常比超长的 `8×1` 或 `12×1` 更容易被图片模型正确生成。
- 对一致性要求较高时，先确定角色标准立绘，再将其作为每个动作的参考图。
- 如果模型无法稳定生成严格 Sheet，可以先生成独立帧，再由本地工具统一缩放、对齐并拼接；播放器仍读取最终规则 Sheet。
- Prompt 中描述的尺寸属于硬性目标，但不能替代程序校验。
