# 多层陪伴空间设计规格

最后更新：2026-07-22

目标版本：`0.5.0`

当前概念图：[`resources/concepts/multi-floor-habitat-concept-v2.png`](../resources/concepts/multi-floor-habitat-concept-v2.png)

第一版概念图保留为构图迭代记录。它成功建立了多层氛围，但在多个房间重复绘制同一角色；第二版明确只保留一名角色，并强化中央电梯、楼梯、楼板和房间之间的明暗分隔，作为当前程序化原型后续调整的视觉基准。

## 产品目标

陪伴空间不再是一间带控制面板的 Q 版单间，而是一座持续运行的建筑剖面。角色是视觉和交互主体，用户主要观察她生活、通过房间与物件介入，并在需要时展开对话或设置浮层。

设计语言借鉴两类方法，但不复制具体作品资产：

- 强轮廓角色：先以剪影、比例、姿态和标志性道具建立辨识度，再补服装与面部细节。
- 可读剖面建筑：楼层、房间功能、连接通道和角色位置一眼可见，行为通过跨房间移动自然表达。

## 第一版建筑

逻辑画布为 `1200 × 900`，包含三层六个主要房间：

| 楼层 | 房间                 | 主要行为                   |
| ---- | -------------------- | -------------------------- |
| 顶层 | 观测室、温室         | 独处、看夜景、环境事件     |
| 中层 | 档案室、实验室、书房 | 长期记忆、研究、学习、对话 |
| 下层 | 厨房、起居室、卧室   | 用餐、自由陪伴、休息       |

楼层通过中央升降井连接。第一版使用节点路径而不是连续寻路：角色从当前房间出口走到连接节点，跨层后再走向目标房间锚点。后续可把节点替换为楼梯、电梯动画或 NavMesh。

## 情境映射

- 自由陪伴 → 起居室
- 一起学习 → 实验室
- 一起吃饭 → 厨房
- 休息一下 → 卧室
- 长期记忆入口 → 档案室
- 低频环境事件 → 观测室或温室

对话中的 `thinking` / `speaking` 临时覆盖当前动作，但不改变所在房间；对话期间切换房间或情境只更新待执行路线，不会提前切回行走表现，对话结束后再继续移动或恢复房间活动。

## 交互层级

1. 点击房间：角色规划路径并移动过去。
2. 点击角色：展开对话浮层，角色停下并转向用户。
3. 点击房间物件：执行该房间内的具体活动。
4. 设置、日程与记忆不永久占用侧栏，分别映射为控制终端、日程板和档案柜。
5. 镜头默认展示整座建筑；空间复杂度增加后再加入跟随角色和手动缩放。

## 动作素材约束

首批运行时动作：`idle`、`walk`、`turn`、`sit-down`、`sit-idle`、`stand-up`、`study`、`thinking`、`speaking`、`eating`、`resting`、`interaction`。

- 所有 Sheet 固定脚底锚点、人物比例、镜头和水平中心。
- 移动动作优先制作左右两个方向；只有服装、发型和道具镜像后仍合理时才运行时翻转。
- 低成本样图先验收纯黑剪影、关键姿态和动作节奏，之后再制作高清母版。
- 概念图不能直接裁切为动作帧，场景图与精灵图是两条独立素材管线。

## `0.5.0` 验收标准

- 至少三层、六个可点击房间。
- 四种现有情境分别映射到不同房间。
- 角色能沿可解释节点路径跨房间、跨楼层移动。
- 对话能打断移动或房间活动，结束后可继续响应情境。
- 场景在普通窗口、最大化和连续调整尺寸时保持完整剖面比例。
- 桌宠与陪伴空间往返不影响页面缩放、角色资源或 PixiJS 生命周期。
- 最终素材缺失时仍可用程序化轮廓原型完成上述交互。
