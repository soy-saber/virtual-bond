# 桌宠尺寸与皮肤系统设计

本文是 Virtual Bond 桌宠表现层的实现基线。目标是采用类似 Custom Knight 的资源替换机制，让符合规范的皮肤可以直接放入指定目录并动态加载，而不需要修改加载器代码。

## 开发顺序

1. 可持久化的桌宠显示比例
2. 开放式皮肤清单与资源校验
3. 通用序列帧动画播放器
4. 皮肤选择、预览和手动重新扫描
5. 动作状态机、打断和回退规则
6. 安全活动范围、低打扰行为和开发模式热重载

## 尺寸模型

- 逻辑画布基准为 `360 × 440`。
- 用户显示比例范围为 `45%–240%`，步长为 `5%`，默认值为 `75%`。
- Electron 窗口和渲染页面使用同一个缩放因子，避免角色、气泡、点击区域和拖拽区域分别计算。
- 缩放时保持桌宠底部位置和水平中心不变，减少角色在桌面上的视觉跳动。
- 桌宠模式锁定窗口最小与最大尺寸；进入陪伴空间后恢复正常窗口缩放和可调整尺寸。
- 比例和窗口位置写入 SQLite 应用设置，重启后恢复。

## 皮肤目录草案

```text
skins/
└─ example-skin/
   ├─ manifest.json
   └─ animations/
      ├─ idle.png
      ├─ walk.png
      ├─ sleep.png
      └─ custom-action.png
```

内置皮肤随应用发布并视为只读；用户皮肤位于应用数据目录。开发模式可额外扫描仓库内的开发皮肤目录并支持重新加载。

## 清单原则

- `animations` 使用开放键名，加载器不维护固定动作枚举。
- 只有 `idle` 是必选动作。
- 缺少目标动作、图片损坏或参数无效时回退到 `idle`；默认皮肤也不可用时显示轻量降级界面。
- 第一版资源使用规则网格 Sprite Sheet，支持单行横向排列以及多行网格。
- 每个动作独立声明文件、单帧尺寸、帧数、FPS、循环方式和可选下一动作。
- 播放器将支持 `columns`、`rows`、`margin` 和 `spacing`，以适配图片模型更容易生成的近方形网格。
- 皮肤级配置声明逻辑画布、脚底锚点、点击区域以及是否允许自动水平翻转。

## 当前实现状态

已完成：

- 纯数据清单解析器，不依赖固定动作枚举。
- `idle` 必选校验以及动画文件存在性检查。
- 拦截绝对路径和 `..` 目录穿越，动画只能引用当前皮肤目录内资源。
- 分层扫描内置、开发和用户皮肤目录。
- 相同皮肤 ID 按扫描顺序覆盖，用户皮肤优先于内置皮肤。
- 无效皮肤单独返回错误，不影响其他皮肤加载。
- 通过受控 IPC 列出皮肤，并可打开应用数据目录下的用户皮肤文件夹。
- 主进程只允许按已扫描的皮肤 ID 和动作名读取 PNG，不向渲染进程开放任意文件路径。
- 读取 PNG 头部的实际尺寸，并根据帧尺寸、行列数、边距和间距核对完整 Sheet 尺寸。
- PixiJS 按规则网格动态切分纹理，支持任意动作名、FPS、循环和 `next` 后继动作。
- 交互动作缺失或加载失败时回退 `idle`；`idle` 也不可用时继续显示原有 CSS 桌宠。
- 自动化测试覆盖合法清单、缺失 `idle`、路径越界、用户覆盖、PNG 头和 Sheet 尺寸。
- 仓库包含首套 `reference-companion` 开发皮肤，可直接验证 PixiJS 播放和 CSS 降级切换。
- Electron 严格 CSP 环境通过 PixiJS 官方静态同步兼容模块初始化，不放宽页面安全策略。
- 桌宠和陪伴空间通过同一播放器加载同一个皮肤动作，仅使用不同画布尺寸、角色比例和脚底位置。
- PixiJS 画布根据设备像素比使用 3×–4× 的内部渲染分辨率，纹理使用线性采样、自动 mipmap 和像素对齐，降低页面缩放和大尺寸显示时的轮廓锯齿与闪烁。
- 桌宠模式把皮肤清单中的 `canvas.hitbox` 映射为人物拖拽区；人物和底部按钮之外的透明窗口区域启用鼠标穿透，窗口放大时不会同步吞掉更大的工作区点击范围。
- AI 生成帧会在透明化后按角色包围盒统一水平中心与脚底坐标，避免锚点漂移被误认为行走。
- 设置页可以播放每套皮肤的 `idle` 预览、持久化当前选择、打开用户皮肤目录并手动重新扫描；切换后桌宠和陪伴空间中的播放器立即同步。
- 重新扫描会列出损坏目录及具体清单错误；已选皮肤被删除或损坏时自动回退到第一套可用皮肤，没有可用皮肤时继续使用 CSS 降级角色。
- 当前选择写入 SQLite 的 `skin.activeId` 设置，换模式和重启应用后继续生效。
- 陪伴空间不再使用固定 `600 × 712` 画布和固定脚底坐标；舞台通过 `ResizeObserver` 获取实际尺寸，播放器只更新渲染器、角色缩放和脚底锚点，不重新读取动画资源。
- 陪伴空间角色保持在文字区右侧的安全区域，脚底始终位于快捷按钮上方；窗口最大化和拖动调整尺寸时，角色大小与站位同步变化。

尚未完成：

- 逐像素 Alpha 轮廓点击区域与自动水平翻转；当前点击区域使用皮肤 `hitbox` 的矩形包围盒。
- 更完整的自主动作调度、随机环境动作与跨动作过渡规则；桌宠侧已先接入 `dragging > speaking > interaction > idle` 的基础优先级。
- 文件变化监听与开发模式热重载。

## 图片生成 Prompt

仓库提供中文模板 [`resources/prompts/sprite-sheet.zh-CN.md`](../resources/prompts/sprite-sheet.zh-CN.md)，用于约束图片模型生成可切分的规则网格 Sprite Sheet。

模板包含：

- 角色、动作、布局和锚点变量。
- 完整中文正向 Prompt。
- 可独立使用的负面约束。
- 八帧行走循环示例。
- 生成后必须执行的尺寸、透明度、帧数和锚点校验。

图片生成结果不能直接视为可信资源。后续播放器和皮肤工具仍需读取实际 PNG 尺寸并校验 Sheet 布局。

首套参考图测试皮肤先使用 [`resources/prompts/reference-neutral-character.zh-CN.md`](../resources/prompts/reference-neutral-character.zh-CN.md) 生成正常站立的标准角色参考，再使用 [`resources/prompts/reference-idle-sprite-sheet.zh-CN.md`](../resources/prompts/reference-idle-sprite-sheet.zh-CN.md) 验证动作模板。正式待机由仓库内 4K 母版本地重导出，锁定骨盆、下半身和脚底：胸肩最大形变约 5 像素，权重峰值位于上胸并向腰部逐渐衰减；头颈保留胸肩 25% 的基础比例，但设置 3 个源像素的最小可见下限，并滞后约 0.75 帧跟随。缩到桌宠画布后头部峰值约 0.75 逻辑像素，颈肩之间平滑过渡。输出为 `4 × 2`、每帧 `1024 × 1024` 的透明 Sheet。可运行 `python scripts/build-reference-idle.py` 重建，不调用图片生成 API。

标准制作流程为：1K 动作模板验收 → 最高 4K 的高分辨率主素材 → 锚点与非动作区域约束 → 1024/512/256 多档运行时导出。完整流程与当前 Provider 限制见 [`image-generation.md`](image-generation.md)。

示例：

```json
{
  "id": "example-skin",
  "name": "示例皮肤",
  "version": 1,
  "canvas": {
    "width": 256,
    "height": 256,
    "anchor": { "x": 128, "y": 240 }
  },
  "animations": {
    "idle": {
      "file": "animations/idle.png",
      "frameWidth": 256,
      "frameHeight": 256,
      "frames": 8,
      "columns": 4,
      "rows": 2,
      "margin": 0,
      "spacing": 0,
      "fps": 8,
      "loop": true
    }
  }
}
```

## 动作与程序能力的边界

新增普通展示动作只需要添加图片和清单项。只有动作需要新的程序能力时才修改业务代码，例如移动窗口、打开界面或调用外部工具。

OpenCode CLI Agent 与普通 DeepSeek 对话保持隔离。Agent 只能提交动作意图，最终动作选择、窗口移动和安全边界由本地状态机决定，外部模型不能直接操纵桌面窗口。

## 后续验收重点

- 调整尺寸、重启和切换显示器后，比例与位置保持正确。
- 任意合法动作名可以加载和播放。
- 缺失非必要动作时稳定回退。
- 损坏皮肤不会阻止应用进入对话。
- 拖拽可以立即打断自主动作。
- 行走不会进入屏幕边缘、用户禁区或主要工作区域。
