# 桌宠尺寸与皮肤系统设计

本文是 Virtual Bond 桌宠表现层的实现基线。目标是采用类似 Custom Knight 的资源替换机制，让符合规范的皮肤可以直接放入指定目录并动态加载，而不需要修改加载器代码。

## 开发顺序

1. 可持久化的桌宠显示比例
2. 开放式皮肤清单与资源校验
3. 通用序列帧动画播放器
4. 动作状态机、打断和回退规则
5. 安全活动范围与低打扰行为
6. 皮肤管理和开发模式热重载

## 尺寸模型

- 逻辑画布基准为 `360 × 440`。
- 用户显示比例范围为 `45%–120%`，步长为 `5%`，默认值为 `75%`。
- Electron 窗口和渲染页面使用同一个缩放因子，避免角色、气泡、点击区域和拖拽区域分别计算。
- 缩放时保持桌宠底部位置和水平中心不变，减少角色在桌面上的视觉跳动。
- 桌宠模式锁定窗口最小与最大尺寸；进入陪伴空间后恢复正常窗口缩放和可调整尺寸。
- 比例和窗口位置写入 SQLite 应用设置，重启后恢复。

## 皮肤目录草案

```text
skins/
└─ example-skin/
   ├─ manifest.json
   └─ animations/
      ├─ idle.png
      ├─ walk.png
      ├─ sleep.png
      └─ custom-action.png
```

内置皮肤随应用发布并视为只读；用户皮肤位于应用数据目录。开发模式可额外扫描仓库内的开发皮肤目录并支持重新加载。

## 清单原则

- `animations` 使用开放键名，加载器不维护固定动作枚举。
- 只有 `idle` 是必选动作。
- 缺少目标动作、图片损坏或参数无效时回退到 `idle`；默认皮肤也不可用时显示轻量降级界面。
- 第一版资源使用横向 Sprite Sheet。
- 每个动作独立声明文件、单帧尺寸、帧数、FPS、循环方式和可选下一动作。
- 皮肤级配置声明逻辑画布、脚底锚点、点击区域以及是否允许自动水平翻转。

示例：

```json
{
  "id": "example-skin",
  "name": "示例皮肤",
  "version": 1,
  "canvas": {
    "width": 256,
    "height": 256,
    "anchor": { "x": 128, "y": 240 }
  },
  "animations": {
    "idle": {
      "file": "animations/idle.png",
      "frameWidth": 256,
      "frameHeight": 256,
      "frames": 8,
      "fps": 8,
      "loop": true
    }
  }
}
```

## 动作与程序能力的边界

新增普通展示动作只需要添加图片和清单项。只有动作需要新的程序能力时才修改业务代码，例如移动窗口、打开界面或调用外部工具。

OpenCode CLI Agent 与普通 DeepSeek 对话保持隔离。Agent 只能提交动作意图，最终动作选择、窗口移动和安全边界由本地状态机决定，外部模型不能直接操纵桌面窗口。

## 后续验收重点

- 调整尺寸、重启和切换显示器后，比例与位置保持正确。
- 任意合法动作名可以加载和播放。
- 缺失非必要动作时稳定回退。
- 损坏皮肤不会阻止应用进入对话。
- 拖拽可以立即打断自主动作。
- 行走不会进入屏幕边缘、用户禁区或主要工作区域。
